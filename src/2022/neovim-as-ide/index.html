Vim as IDE 2.0

2022.04.03

想起第一次看到别人在用 Vim 的时候，还是第一家公司看到一周在用的时候，第二次看到直接使用 Vim 作为前端 IDE 的时候是看到瑾源在用的时候（看到的时候感觉十分的震撼，因为自己 VSCode 配上 VSCode 的 vim 插件就相当不错了，基本能满足日常开发了，从没想过还能直接上 Vim 作为 IDE

在今年 1 月 16 号写完 VIM setup as an IDE 这篇文章后，使用 Vim 作为前端 IDE 开发的时候，还是觉得有点不顺，于是配置好的 3 天后基本就放弃了，主要是那时候几个问题导致的，

  • 使用 tmux window/ session 和 vim tab 的不熟练
  • 每次 git commit 前会想要看下各个文件的改动，在 VSCode 里使用 gitlens 插件就行了，但是在 Vim 里暂时没看到什么办法
  • VSCode 里下方的命令行的 toggle
  • Vim 里不会自动保存，需要使用 ：w 写保存 buffer，让自己没有安全感

但是使用 VSCode 的时候还是有几个问题让我有点不舒服，所以准备再次尝试下，

  • 使用 VSCode 里的目录树打开文件不是特别好打开虽然也使用了 Vim 快捷键的插件
  • toggle 命令行的快捷键有点难记但是又不想 remap。。。
  • 启动速度慢（就没有达到 sublime 或者 vim 那种丝滑的感觉
  • VSCode 里折叠函数的时候，如果使用 Vim 方向键 cross 这个折叠区域的时候，整个折叠函数会自动打开
  • VSCode 的 Vim 插件总感觉不能完全实现 Vim 的所有功能
  • 需要不时使用鼠标来剪贴和滚动查看代码觉得有点手疼

这一次上面提到的所有问题都在 Vim 和 Tmux 里解掉了。

先说下感想吧，使用了一个礼拜进行项目开发感觉还是十分的流畅的，因为强迫使用纯键盘来 Coding and Navigation 所以逼自己将组件颗粒度细化了，不然组件代码太长看上去很累，需要不断的 ctrl-d 来滚动。

好了，接下来就来看下我的 Vim 环境配置以及遇到的一些问题和解法吧。

遇到的问题和解法

  1. Terminal toggle。使用 prefix " 命令在下方创建一个命令行，切换到 vim 所在的 tmux pane 然后使用 prefix z 来 toggle 命令行。
  2. Folding。zc 关闭折叠，zo 打开折叠。
  3. Indent。进入 Vim visual mode 然后，使用 < 或者 > 然后可以配合 . repeat 键。
  4. 在 nerdtree 里创建文件。使用 m 唤起操作菜单。
  5. Git unstage。:G 然后使用 u 就可以 unstage 一个文件。
  6. 将搜索的文件新开 tab。使用 ;f 来搜索项目文件，ctrl-p 和 ctrl-n 上下移动文件，对于选中的文件可以使用 ctrl-u 来新开 tab。
  7. 全局文件内容全局搜索。;r 来使用fzf vim 进行全局搜索。
  8. Git commit 前看下文件改动。使用 :G 查看所有变动的文件，然后可以使用 ctrl-n 或者 ctrl-p 来将光标上下移动，然后使用直接敲击 dv 命令就可以以分屏的方式查看当前光标所在行（文件）的 diff 了，其他的一些快捷键可以参考下官方文档。可以使用 te 回车来新建一个 vim tab 进行 diff review。
  9. 系统异常了文件丢失了怎么办？可以去 /tmp 目录下找对应的 .swp 文件进行恢复。
  10. vim 里 ctrl-d 不会翻页怎么办，而是会删除。这里是 Mac 系统设置了大小写键绑定成了 control 键，开了 karabiner 会有这个问题，需要在 karabiner 里也将 大小写键绑定成了 control 键。（Tips: 可以通过 karabiner-eventviewer 排查问题）
  11. 使用 tmux 同时开发多个项目，可以使用 tmux session，每一个 sessioin 作为管理一个项目。使用 tmux 命令会默认创建一个 session，你也可以使用 tmux new -s mysession 来创建命名后的 session。
  12. 运行后台服务，比如 wds。可以使用 tmux window，可以使用 <prefix> c 来创建一个 window，用来跑服务。但是目前我比较喜欢类似 VSCode 里的 terminal 的布局，所以我会使用 <prefix> " 在屏幕下方创建一个 tmux pane，然后使用 <prefix> l 移动到上面的 pane，然后使用 <prefix> z 来全屏上面的 pane，这样下面的 pane 就会自动 minimized 了，prefix 我定义为了 ctrl+space，然后你也可以使用 tmux window 临时打开一些项目。
  13. 折叠。可以使用 zc 和 zo 来进行 fold 和 unfold，这里配置了 set foldmethod=indent 是根据缩进来进行 fold 的，所以相同的缩进的行会被 fold，还是很方便的。而且缩进后会变成一行，你使用 vim 方向键上下移动而会将其视作一行，而不会移到该行自动展开，非常 nice！
  14. 打开 nerdtree，快速定位到当前打开文件所在的位置，使用 <localleader>e 就行了。我将 <localleader> 和 <leader> 都设置为了 <space>。
  15. Vim pane 之间的移动可以使用 ctrl-hjkl，Tmux pane 之间的移动可以使用 <prefix>-hjkl 或者 <prefix>-<Tab>，Vim 的 tabs 功能可以使用 tab 键进行切换，如果你只有一个 tab 的话，tab 键还可以用来切换 vim pane。
  16. 重命名 tmux window。使用 <prefix> ,。
  17. 刷新 nerdtree，使用 r，toggle 使用 ctrl-e。
  18. 使用 zR 和 zM 来打开当前 vim buffer 里的所有的 fold 和关闭所有的 fold。
  19. 可以使用 te 命令创建一个新的 vim tab 然后 :G 来查看改动，review 之后使用 <prefix>z toggle 我们之前创建的 tmux pane，commit 完之后可以使用 tq 关闭当前的 vim tab。
  20. 将当前的 vim pane 移动到新的 tab，有时候因为当前 tab 已经分屏了，但是在工作的那个 vim pane 需要继续分屏来开发，这样可以使用 ctrl-w-T 来实现，然后继续 sv 进行 vim 分屏。
  21. 找到当前引用该组件的地方，可以使用 coc 的 coc-references 功能，这里将其 map 成了 gr。
  22. 可以使用 ;r 进行 rg 的搜索，' 加上搜索词进行 exact match，当然还可以进行 prefix 匹配等，语法可以参考这里。
  23. 你可以使用 :Gclog 来查看 commits 的改动，或者使用 fzf 的 ;c 命令来查看 commits 的改动（;c 是自己设置的快捷键）。
  24. FzF 里使用 :Lines 等也支持 rg 的匹配符。
  25. 你还可以使用 :Colors 来切换一次性的主题。
  26. 默认的 tmux 分屏是分一半，你可以使用 <prefix> 配合 l r u d 来 resize。l 代表 left，其他同理。
  27. 临时格式化查看一个文件，可以通过 te 命令直接创建一个 vim buffer（tab）然后输入文件名带后缀，然后在 visual mode 选中所有行之后使用 coc 的 prettier 命令 <leader> fp 来格式化。
  28. 全替。这个难度稍微高一点，需要学习下 vim 的正则。单文件替换的话可以使用 %s/old/new/gc 来替换，这里的 /c flag 用来替换前做一次确认。项目纬度的全局文件替换的话，可以使用 :vimgrep old * 来进行全局搜索，匹配的文件会被加载到 quickfix list 里去，可以使用 :copen 进行列表查看，真正替换的时候，使用 :cfdo %s/old/new/gc 进行替换，替换完成后真正写入 buffer 需要使用 :cfdo update 进行替换（方法来自 Practical Vim 这本书的 Tip 97），单文件替换的话，感觉一般使用 coc 插件的 <leader>rn 进行重命名就行了。vim 里 exact match 的话，可以看下这里，\<stringtofind\>，将要搜索的放到两个转移后的箭头里面即可，这里的 \< 表示 match 开头，\> 表示 match 结尾。- 2022.04.05
  29. 删除多行末尾的 ; 分号，在 VSCode 里可以使用 multi-cursor，在 Vim 里可以使用 vim 的 macro，首先使用进入 normal mode，输入 q 开始录制 macro，然后键入 a 来指定 register 为 register a，然后我们先输入 $x 跳到行末对单行的 ; 进行 fix，然后使用 q 完成对单行 fix 的命令录制，然后进入 visual mode 选中还需要进行 fix 的行，然后输入 :normal @a replay register a 里录制的 macro，它会对每一行进行 fix，每一行之间的 fix 都是独立的，也就是某一行 fix 报错了，也不会影响到其他行的 fix，也被称为 parallel fix。还有一种是 serial fix，也就是某一行 fix 报错了，接下来的行不会继续 fix，你直接使用 count@a 即可，如果有 22 行要 fix 的话，那么使用 22@a。（方法来自 Practical Vim 的 macro 章节）。在 Vim 里 multi-cursor 其实真的不需要，可以看下这篇文章，这里的 visual block 模式和 multi-cursor 就很像，我们可以使用 visual block mode 来实现下，首先使用 ctrl-v 在当前光标下进入 visual mode，然后你可以使用 hjkl 来进行多行选中，多选一行相当于多添加了一个光标，然后使用 j 方向键选中所有的 ;，然后使用 x 删除 ;，这样所有的 ; 就会被删除了。如果你想要插入字符的话，只有在你退出插入模式进入 normal mode 的时候，单行的 change 才会反应到多行上面。- 2022.04.05
  30. 对于 vim 里不熟的命令可以使用 :help command 来查看命令，比如，:help ~ 来查看 ~ 的使用方法。- 2022.04.06
  31. 在 insert mode 的时候使用 ctrl-w 来替换删除键。- 2022.04.06
  32. 删除全部的 console.log，还是发现了全替的使用场景。首先使用 ;r 输入 console.log( 来搜索所有的 console.log，然后使用 fzf 的 ctrl-a 全选，然后使用 fzf 的 ctrl-q 将所选全部 load 到 vim 的quickfix list 中去，这样我们就可以全替了，但是在全替之前我们得对这个 quickfix list 做一次筛选，因为有的 console.log 是跨了几行的，我们可以使用 vim 内置的 Cfilter 插件对 quickfix list 做一次筛选，使用 :Cfilter /console\.log(.*);/ （并使用了正则）来过滤，过滤完了使用 cdo s/console\.log(.*);//c 进行确认式替换单个 quickfix entry，最后替换完成后使用 cdo update 即可写入所有的 vim buffer。最后使用 ;r 手动替换剩下的几个即可。（备注：可了解下 :cdo, :cfdo, 以及 quickfix list），对了你还可以使用 :copen 来查看 quickfix 列表。(还有一种思路可以试一下 multi-line match）- 2022.04.06
  33. 删除全部的 console.log 方法二，不使用 Rg（ripgrep），还是使用 vimgrep，这里的难点其实就是不搜索 .gitignore 里 ignore 的文件即可。可以看下这个解决方案，可以使用 :vimgrep /console\.log(.*);/g `git ls-files` 来将 console.log 加入到 quickfix list 去，省去了上面的 filter 的步骤。最后再使用 rg 来处理剩下的几个多行的 console.log。为了方便在 vimrc 里加了个自定义命令 command -nargs=1 Gvimgrep vimgrep /<args>/g `git ls-files` | copen, 可以使用 Gvimgrep 直接进行搜索，比如，:Gvimgrep console\.log(.*); （其实替换的这一步的话也可以使用 register 来 dd 一下）- 2022.04.06

Vim 相关

Vim 中主要的操作是，operator+motion/text object 配合 count。

motion 和 text object 差不多，但是 motion 是单向的，而 text object 是双向的。举个例子，motion w 是跳到下一个 word，是向右的，而 text object iw 是表示当前光标所在的单词，是双向的表示一个范围，和 operator 配合起来，举个例子，dw 删除直到下一个单词，而 diw 是删除当前的单词，下面是一些常用的 operator，motion 和 text object，

  • 比较常用的 operator: c, d, y,gu,gU
  • 比较常用的 motion: h,j,k,l,0,$,f,F,t,T,;,,,w,e,ge
  • 比较常用的 text object: aw, iw,at,a/i`'"{<[(。(a for around, i for inner)

上面的 operator+motion 模式是对于 normal mode 来说的，在 visual mode 中的话，是 motion+operator 模式，比如 vegU。

  1. 之前不是很熟悉 operator 的时候都是使用 VegU 先进入 visual mode 然后使用 motion e 配合 gU 进行大写转换，现在发现可以直接使用 gU operator，比如，gU 配上 e motion，也就是 gUe 来完成。
  2. Vim 里有 sessions，tabs，windows 的概念，Tmux 也有 sessions，windows（tabs），panes（splits） 的概念，这三个概念都是一一对应的， sessions 对 sessions，tabs 对 windows，windows 对 panes。目前的工作流中，使用了 vim 的 tabs 和 windows 以及 tmux 的 sessions 和 panes。vim 的 tabs 用来看不同的文件，有需要的话会进行分屏也就是 windows 的概念，因为开发过程中需要跑开发服务器放在后台，所以会使用 tmux 的 panes 来进行上下分屏在下面开一个 terminal，然后使用 tmux 的 sessions 来进行不同项目的管理。
  3. Vim 也可以直接调出 termnial 使用 :termnial 的命令就可以了，然后使用 i 来和 terminal 进行交互，使用 ctrl-\ ctrl-n 来回到 vim 编辑器。但是这个没有 tmux 方便。
  4. ? 是反方向搜索，相对于，/。
  5. Register。用来进行复杂的操作，可以录制一堆 operator+motion，然后 replay。使用 q 开始煮呗录制，选择 a-z 中的 26 一个字母进行录制（也就是选择 register），录制完毕敲击 q 结束录制。使用 @ 加上你选的字母就行 replay（serial 的，如果对多个文件进行回放的时候报错的话，1个文件报错那么就不会继续了），所以你也可以使用 :normal @a 进行 replay（parallel 的，一个文件报错不会影响其他文件的 replay）。
  6. 插入模式下使用 CTRL-O 可以使用一次 normal mode 的命令，使用完成自动切换为插入模式。
  7. ctrl-z 可以将当前进程放入后台，使用 fg<Enter> 可以重新加载当前进程。
  8. vim command line mode 下可以使用 <C-d> 来调出备选项，以及使用 <Tab> 来进行补全。
  9. 使用 :! 可以执行 shell 等的外部程序命令，比如 :!ls。
  10. % 可以用来 navigate 匹配的括号和 html 标签等。
  11. 替换。s 替换光标所在行，%s 替换光标所在文件。
  12. K 可以查看命令文档，:help yourcommandtosearch 速查命令。
  13. Marks。可以使用 |``|` 或者 ' 进行 mark 的跳转，可以使用 :marks 来查看所有的标记。'0 可以跳转到最新的一次变更。(不怎么常用，一般使用默认的 Marks，比如 ctrl-o 和 ctrl-i 命令来跳转到默认的一些 marks 标记。
  14. Buffers。可以使用 :ls 查看所有的 buffer。
  15. Undotree。可以使用 :undolist 来查看当前文件的 change list。- 2022.04.07

其他

  1. Defx 使用报错的话，可以在安装完依赖后，跑下 :UpdateRemotePlugins 命令，并且它使用的是 python3。
  2. 中文 yank 乱码。在 zshrc 里加了行 export LANG="UTF-8"并且在 vim 配置里加入了 lang en_US.UTF-8。
  3. Coc 插件不正当删除后报错。类似 [coc.nvim] Unable to load global extension at /Users/cary/.config/coc/extensions/node_modules/coc-cssmodules: package.json not found，可以删除 ~/.config/coc 文件夹后重新进入 vim 自动安装 coc 插件，参考了下这里。
  4. Github 的代码补全插件配置完成需要跑下 :Copilot setup。
  5. Iterm2 可以设置字体 ligatures，目前使用 Monolisa 字体。
  6. 可以在系统里设置键位 repeat 为最大，delay 最大减一，提高移动速度。
  7. Coc.nvim 安装出问题，可以切到 cd ~/.local/share/nvim/plugged/coc.nvim 这个目录下，然后 npm i。
  8. 进入 tmux 后使用 zsh，输入 ls 会变成 lss，排查发现原来是它的 zsh-autosuggestions 的插件的问题。。。，然后删除键还会变成空格键，最后在 zsh 配置里加了 TERM="xterm" 两个问题都好了。
  1. 可以使用 :hi 命令查看 vim 的配色，从而进行自定义。
  2. Vim 在进入 visual mode 的时候，选择的时候光标的颜色和选择的高亮颜色不一致很突兀，可以在 iterm 里改下 Cursor 和 cursor text 两个属性。还有光标的高度和选择的字符的高度不完全一致，发现可以设置 iterm text 里的行高。（n/n 设置为了 100 目前， v/i 也设置为了 100 当前字体下）
  3. 改了 vim 配置之后可以使用 :source ~/.vimrc 进行配置的重新加载。
